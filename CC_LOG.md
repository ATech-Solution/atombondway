### High-Level Logic

```
User visits /{locale}/page
  → middleware.ts (next-intl) detects/redirects to correct locale
  → (frontend)/[locale]/layout.tsx fetches global data from Payload (server component)
  → Page component fetches page-specific data from Payload (server component)
  → Section components render with props
  → Client components (Header, ContactForm, LocaleSwitcher) handle interactivity

Admin visits /admin
  → Payload admin panel (excluded from next-intl middleware)
  → JWT auth (stored in cookie)
  → Admin edits collections/globals
  → Data stored in SQLite at data/payload.db
  → Media uploaded to public/media/

Contact form submission
  → ContactForm.tsx (client) → POST /api/contact
  → route.ts validates input → nodemailer sends email
  → Dev: Mailpit (localhost:1025) | Prod: AWS SES
```

## Entry 004

**Timestamp**: 2026-04-03T10:00:00Z  
**User Intent**: Fix admin hydration error (`<html>` nested in `<body>`) and populate all content from atombondway.com.

### Hydration Fix
- **Root cause:** `app/layout.tsx` rendered `<html><body>` and Payload's `RootLayout` (used in `(payload)/layout.tsx`) also rendered its own `<html><body>`, causing nested invalid HTML.
- **Fix:** `app/layout.tsx` is now a bare pass-through (`return <>{children}</>`). Each route group provides its own HTML shell:
  - `(payload)/layout.tsx` → Payload's `RootLayout` (owns `<html><body>`)
  - `(frontend)/[locale]/layout.tsx` → owns `<html lang={locale}><body>` with NextIntlClientProvider, Header, Footer
